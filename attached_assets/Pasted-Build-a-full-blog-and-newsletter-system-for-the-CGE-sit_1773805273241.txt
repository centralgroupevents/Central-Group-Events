DATABASE — add these new tables:

posts:
- id (serial primary key)
- title (text, required)
- slug (text, unique, auto-generated from title)
- excerpt (text)
- content (text — stores rich text HTML)
- coverImageUrl (text, nullable)
- isPublished (boolean, default false)
- isGated (boolean, default false)
- publishedAt (timestamp, nullable)
- createdAt (timestamp, default now)
- updatedAt (timestamp, default now)

post_versions:
- id (serial primary key)
- postId (integer, references posts.id)
- title (text)
- content (text)
- savedAt (timestamp, default now)
- savedBy (integer, references admin_users.id)

post_views:
- id (serial primary key)
- postId (integer, references posts.id)
- viewedAt (timestamp, default now)

link_clicks:
- id (serial primary key)
- postId (integer, references posts.id)
- url (text)
- clickedAt (timestamp, default now)

admin_users:
- id (serial primary key)
- email (text, unique)
- passwordHash (text)
- role (text — "superadmin" or "editor")
- inviteToken (text, nullable)
- inviteAccepted (boolean, default false)
- createdAt (timestamp, default now)

Update subscribers table to add these columns if they don't exist:
- referrer (text, nullable)
- hasAccess (boolean, default true)
- createdAt (timestamp, default now)

BACKEND API ROUTES:

Posts:
- GET /api/posts — return all published posts sorted by publishedAt desc. Include id, title, slug, excerpt, coverImageUrl, isGated, publishedAt
- GET /api/posts/:slug — return full post by slug including content, isGated, isPublished
- POST /api/posts — create post (admin/editor only)
- PUT /api/posts/:id — update post, save a snapshot to post_versions before updating (admin/editor only)
- DELETE /api/posts/:id — delete post (superadmin only)
- PATCH /api/posts/:id/publish — toggle isPublished (superadmin only)
- PATCH /api/posts/:id/gate — toggle isGated (superadmin only)
- GET /api/posts/:id/versions — return all versions for a post (admin only)
- POST /api/posts/:id/restore/:versionId — restore a post to a previous version (admin only)

Subscriber gating:
- POST /api/subscriber/check — accepts { email, referrer }. Check if email exists in subscribers table. If found, return { access: true }. If not found, insert the email into subscribers table with the referrer, then return { access: true, isNew: true }. Set a 30-day cookie called "cge_subscriber" with the email value (httpOnly, signed).
- GET /api/subscriber/verify — checks if the "cge_subscriber" cookie is valid and the email exists in subscribers table. Returns { access: true/false }.

Link click tracking:
- GET /go — accepts ?url= and ?postId= query params. Log the click to link_clicks table then redirect to the url. Example: /go?url=https://eventbrite.com&postId=3

Analytics:
- GET /api/analytics — return: { totalSubscribers, postViews: [{ postId, title, views }], linkClicks: [{ url, count }], memberSources: [{ referrer, count }] } (superadmin only)

Admin users:
- POST /api/admin/invite — send invite email to a new editor (superadmin only). Generate an inviteToken, store it, send email with link to /admin/accept-invite?token=xxx
- POST /api/admin/accept-invite — accepts { token, password }. Find admin_user by inviteToken, hash password, set inviteAccepted: true, clear token
- POST /api/admin/login — accepts { email, password }. Return JWT stored in httpOnly cookie.
- POST /api/admin/logout — clear the auth cookie
